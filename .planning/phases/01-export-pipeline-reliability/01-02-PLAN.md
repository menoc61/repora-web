# Plan 01-02: Add LibreOffice + assets to backend Docker image

## Objective
Make the backend Docker image capable of producing high-quality PDFs via LibreOffice conversion, and include the `public/assets` directory so cover images are available at runtime.

## Context
- `backend/Dockerfile` uses `node:22-alpine` and copies only `dist` (+ package.json, drizzle.config.ts, entrypoint). It never copies `public/`, so the cover images from Plan 01-01 are missing in-container.
- `docxToPdf.ts` already detects LibreOffice at `/usr/bin/libreoffice` / `/usr/local/bin/libreoffice` and `export.service.ts:72` uses it when available, else falls back to PDFKit.
- Alpine has no LibreOffice in main repos. Switch to a Debian-slim base so `apt-get install` works.

## Tasks

<task>
<read_first>
  - backend/Dockerfile
  - docker-compose.yml
  - backend/src/services/docxToPdf.ts
</read_first>
<action>
1. Change base image from `node:22-alpine` to `node:22-bookworm-slim` for the `runner` (and `builder`/`deps` if they need native builds ‚Äî keep `deps`/`builder` on `node:22-bookworm-slim` too for consistency).
2. In the `runner` stage, before `EXPOSE`, install LibreOffice headless writer:
   - `RUN apt-get update && apt-get install -y --no-install-recommends libreoffice-writer libreoffice-calc fonts-liberation && rm -rf /var/lib/apt/lists/*`
   - Note: `libreoffice-writer` pulls core libs; if conversion needs more, use `libreoffice` (full). Prefer the smaller `libreoffice-writer` first.
3. Copy the shared assets into the image so Plan 01-01's resolver finds them at `/app/public/assets`:
   - `COPY public/assets ./public/assets`  (from repo root context into /app/public/assets)
4. Verify `docker-compose.yml` mounts/build context still build the backend from the repo root (so `public/assets` COPY resolves). Adjust `build.context` if it currently points only at `backend/`.
5. Confirm the entrypoint still runs (`docker-entrypoint.sh` unchanged).
</action>
<acceptance_criteria>
  - Backend image builds successfully on `node:22-bookworm-slim` with LibreOffice present.
  - Inside the container, `which libreoffice` (or `soffice`) resolves to a binary `docxToPdf.findLibreOffice()` detects.
  - `/app/public/assets/cover_bg.png` exists in the built image.
  - `docker-compose.yml` build context includes repo root so the `public/assets` COPY works.
</acceptance_criteria>
<verify>
  - `docker compose build backend` succeeds.
  - `docker compose run --rm backend which soffice` returns a path.
  - `docker compose run --rm backend ls /app/public/assets/cover_bg.png` returns the file.
</verify>
</task>

## Success Criteria (this plan)
- Docker backend has LibreOffice + assets, enabling real PDF conversion and cover images.
