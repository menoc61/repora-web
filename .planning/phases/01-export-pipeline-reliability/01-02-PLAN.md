# Plan 01-02: Keep backend image light + include assets

## Objective
Keep the backend Docker image small (Alpine-based, no LibreOffice) while ensuring the `public/assets` directory is available at runtime so cover images render. PDF export uses the lightweight built-in PDFKit fallback in Docker; LibreOffice remains an optional, mountable upgrade for higher-fidelity PDFs.

## Context
- `backend/Dockerfile` used `node:22-alpine` and copied only `dist`. It never copied `public/`, so cover images were missing in-container.
- Baking LibreOffice into the image adds 400MB–1GB+ and contradicts the "light, optimized build" goal. Rejected.
- `docxToPdf.ts` already detects LibreOffice if present (`isLibreOfficeAvailable()`) and `export.service.ts:72` falls back to `generatePdfFallback` (PDFKit) otherwise. PDFKit is already a dependency and lightweight.
- Backend build context is now the repo root (see docker-compose.yml) so `COPY public/assets ./public/assets` resolves.
