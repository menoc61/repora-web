# Plan 01-01: Fix DOCX cover image path resolution

## Objective
Make the DOCX cover/back-cover background images resolve correctly both in dev and inside the Docker container, so the cover page renders with its navy background image instead of silently dropping it.

## Context
- `backend/src/services/exportDocx.ts:14` defines `PROJECT_ROOT = path.resolve(process.cwd(), '..')`.
- In dev, `cwd = backend/` → resolves to repo root → `public/assets/cover_bg.png` exists ✓.
- In Docker, `cwd = /app` → resolves to `/` → `/public/assets/cover_bg.png` missing ✗ (image silently skipped by `loadImageBuf`).
- The images actually live at repo-root `public/assets/` (shared with frontend), never under `backend/`.

## Tasks

<task>
<read_first>
  - backend/src/services/exportDocx.ts
  - public/assets/cover_bg.png
  - public/assets/backcover_bg.png
</read_first>
<action>
1. Replace the `PROJECT_ROOT` constant (line 14) with a robust asset resolver:
   - Add a helper `function resolveAsset(name: string): string | null` that tries, in order:
     a. `path.join(process.cwd(), 'public', 'assets', name)`  (Docker: cwd=/app → /app/public/assets/*)
     b. `path.join(process.cwd(), '..', 'public', 'assets', name)`  (dev: cwd=backend → repo/public/assets/*)
     c. `path.join(__dirname, '..', '..', 'public', 'assets', name)` (built dist/services → backend/public — fallback)
   - Return the first candidate that `fs.existsSync`.
2. Update line 368: `loadImageBuf(path.join(PROJECT_ROOT, 'public', 'assets', 'cover_bg.png'))` → `loadImageBuf(resolveAsset('cover_bg.png') ?? '')`.
3. Update line 509 (back cover): `loadImageBuf(path.join(PROJECT_ROOT, 'public', 'assets', 'backcover_bg.png'))` → `loadImageBuf(resolveAsset('backcover_bg.png') ?? '')`.
4. Ensure no other references to `PROJECT_ROOT` remain (grep). Remove the constant if unused.
</action>
<acceptance_criteria>
  - DOCX export builds without throwing when assets are present or absent (graceful null, not a crash).
  - In Docker (cwd=/app) with assets copied to /app/public/assets, the cover background image is embedded.
  - No remaining usage of the broken `PROJECT_ROOT` pattern for asset resolution.
</acceptance_criteria>
<verify>
  - Build backend (`npm run build` in backend) succeeds.
  - Run `node -e` check: `resolveAsset('cover_bg.png')` returns a path that `fs.existsSync` is true for in the repo dev layout.
</verify>
</task>
