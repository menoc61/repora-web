# Plan 01-03: Make DOCX cover title reflect document type

## Objective
Replace the hardcoded "CAHIER DES CHARGES" cover title with the title derived from the document's selected type (from onboarding config), so a "Rapport Technique" or "Specification Fonctionnelle" produces the correct cover label.

## Context
- `exportDocx.ts:24` already accepts `documentType?: string` and line 299 selects the template via `getTemplate(doc.config.documentType || 'cahier_des_charges')`.
- `docTemplates.ts` `DOCUMENT_TEMPLATES` has `id` + `name`: `cahier_des_charges`→"Cahier des Charges", `rapport_technique`→"Rapport Technique", `spec_fonctionnelle`→"Specification Fonctionnelle".
- Lines 385-390 hardcode two `TextRun`s: "CAHIER DES" / "CHARGES". Need to derive an uppercase title from the template name and split into 1-2 lines by length.
