# 02-01: Refactor S3 service to init client/module once

**Requirement:** INF-01
**Goal:** Cache the `@aws-sdk/client-s3` module reference in addition to the client, so
the dynamic import runs once instead of per-call (`ensureBucket`, `uploadExport`,
`downloadExport` each re-import today).

**Files:** `backend/src/services/s3.service.ts`

**Changes:**
- Add module-scope `let _s3mod: any = null`.
- Add `async function loadModule()` that imports once and caches.
- `getClient()` calls `loadModule()` then builds `_client` once.
- `ensureBucket`/`uploadExport`/`downloadExport` call `loadModule()` instead of re-importing.

**Verification:** `npx tsc --noEmit` in backend; dynamic import still guarded so local dev
(without the package) does not crash.
