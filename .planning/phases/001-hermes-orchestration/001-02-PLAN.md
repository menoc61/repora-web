<objective>
Implement the Hermes negotiation loop (accept/rescope/adjust) between pipeline stages, integrate template-based document generation, and enhance SSE streaming with GenerationContext events. Refactor the orchestrator from a single flat function in hermes.ts into a modular pipeline directory.

Purpose: Currently the orchestrator blindly accepts every agent's output and proceeds — there is no quality gate between stages. The Writer could generate garbage and Hermes would still dispatch UML and Reviewer on top of it. This plan adds the negotiation layer that makes the multi-agent system a true *orchestrator* rather than a sequential script.

Output:
- `backend/src/ai/pipeline/orchestrate.ts` — modular orchestration with negotiation
- `backend/src/ai/pipeline/negotiate.ts` — accept/rescope/adjust logic + G2 loop breaker
- `backend/src/ai/hermes.ts` — reduced to runAgent + generation tracking + exports
- `backend/src/services/template.service.ts` — template fetching for generation
- `backend/src/services/outline.service.ts` — template outline merging
- `backend/src/routes/projects.ts` — template-aware generate endpoint

All existing behavior preserved. The 5-stage pipeline order (Planner → Writer → UML → Tables → Reviewer) does not change.
</objective>

Create `backend/src/ai/pipeline/negotiate.ts`:

Define and export `NegotiationDecision` discriminated union:
```ts
type NegotiationDecision =
  | { decision: 'accept'; reason: string }
  | { decision: 'rescope'; feedback: string; sectionId?: string }
  | { decision: 'adjust'; key: string; value: unknown; reason: string }
  | { decision: 'abort'; reason: string }
```

Define and export four functions:

1. `acceptHandoff(agentName: string, outputSummary: string): NegotiationDecision`
   - Called after each agent completes (before proceeding to next pipeline stage / next section).
   - For Writer: trivially accepts for now (the Reviewer is the quality gate). The accept decision means "proceed to next section / next agent."
   - Returns `{ decision: 'accept', reason: `${agentName} completed successfully` }`.

2. `rescopeHandoff(agentName: string, sectionId: string, feedback: string, rescopeCount: number): NegotiationDecision`
   - **G2 guardrail (loop breaker):** If `rescopeCount >= 3`, return `{ decision: 'abort', reason: 'max rescope attempts exceeded for section ${sectionId}' }` and the section is marked `needs_human_review`.
   - Otherwise, return `{ decision: 'rescope', feedback, sectionId }` so the orchestrator can re-invoke the Writer with the feedback injected into the prompt.
   - Called when the Writer's output fails a quality heuristic (see Task 2).

3. `adjustContext(ctx: GenerationContext, key: string, value: unknown): GenerationContext`
   - Mutates `ctx.metadata[key] = value`.
   - Returns `ctx` (for chaining).
   - Called when downstream agents need corrected upstream context (e.g., Planner outline had a structural error discovered during writing).

4. `evaluateWriterOutput(sectionContent: string): { passed: boolean; issues: string[] }`
   - Heuristic quality checks (no LLM — pure code, < 50ms):
     - Content length < 50 chars → `{ passed: false, issues: ['content too short'] }`
     - Content matches placeholder pattern (contains only whitespace, "Lorem ipsum", or the section title repeated) → `{ passed: false, issues: ['placeholder content detected'] }`
     - Content is all English when French was requested (check for common French words: "le", "la", "les", "des", "une", "est", "dans", "pour") — if none found → `{ passed: false, issues: ['content may not be in French'] }` (warning, not hard fail)
   - Otherwise → `{ passed: true, issues: [] }`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Planner output → outline parser | LLM-generated JSON may be malformed, missing keys, or injection — validated by isOutlineJson() before use |
| Writer output → evaluateWriterOutput | Untrusted text from LLM — heuristic checks are defensive, not authoritative |
| Template outline → Planner prompt | Template data from DB is trusted (admin-created), but JSON structure is validated before merging |
| SSE stream → client | Event data is JSON-serialized — no HTML injection risk since client parses via JSON.parse |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-001-04 | Denial of Service | rescope loop | mitigate | G2 guardrail: rescopeHandoff aborts after 3 attempts per section. Rescope count tracked in GenerationContext.rescopeCount Map. |
| T-001-05 | Elevation of Privilege | Template outline injection | mitigate | Template outlines are admin-created documents — the mergeTemplateWithOutline function validates structure before merging. Malformed template outline → fallback to Planner-only output. |
| T-001-06 | Information Disclosure | SSE stream events | accept | SSE events contain section titles and agent names only — no PII, no API keys, no project briefs. Low risk. |
| T-001-07 | Repudiation | Pipeline stage tracking | accept | Pipeline stage events are informational only — no audit requirement. Audit logging for document generation already exists in projects.ts route via logAudit(). |
| T-001-SC | Tampering | npm packages | mitigate | Same as Plan 001-01 — verify via npm audit in CI. No new dependencies added in this plan. |
