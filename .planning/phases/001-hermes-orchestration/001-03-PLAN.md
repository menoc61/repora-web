**Define HermesEvent types for frontend:**

```ts
// ── Hermes SSE Event Types ──
export interface HermesAgentStatus { type: 'agent_status'; agent: string; status: string; section?: string; section_title?: string; sections_created?: number; [key: string]: unknown }
export interface HermesToken { type: 'token'; token: string; agent: string }
export interface HermesToolCall { type: 'tool_call'; agent: string; tool: string; args: unknown }
export interface HermesToolResult { type: 'tool_result'; agent: string; tool: string; result: unknown }
export interface HermesSectionComplete { type: 'section_complete'; section_id?: string; title?: string; [key: string]: unknown }
export interface HermesContextUpdated { type: 'context_updated'; agent: string; key: string; value: unknown }
export interface HermesGenerationError { type: 'generation_error'; agent: string; message: string; recoverable: boolean; sectionId?: string }
export interface HermesDone { type: 'done'; document_id?: string; agent?: string; [key: string]: unknown }
export type HermesEvent = HermesAgentStatus | HermesToken | HermesToolCall | HermesToolResult | HermesSectionComplete | HermesContextUpdated | HermesGenerationError | HermesDone
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SSE stream → React state | Untrusted JSON from server parsed into HermesEvent — type narrowing via discriminated union on `event.type` prevents injection |
| User prompt input → generate endpoint | Free-text input sent as POST body — Express body parser handles sanitization; no HTML rendering of untrusted prompt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-001-08 | Spoofing | SSE event source | accept | SSE endpoint is same-origin (same Express server). No cross-origin SSE without explicit CORS config. Client connects to known backend URL. |
| T-001-09 | Tampering | HermesEvent parsing in frontend | mitigate | Discriminated union on `event.type` ensures malformed events (missing fields, wrong types) are caught by TypeScript at compile time and ignored at runtime. |
| T-001-10 | Information Disclosure | Agent progress in UI | accept | Agent chip metadata (agent names, section titles) is not sensitive — same data visible in server logs. No PII exposure. |
| T-001-SC | Tampering | npm packages (react, @tanstack/react-query) | mitigate | Standard React ecosystem — verify via npm audit in CI. No new dependencies added. |
</threat_model>
