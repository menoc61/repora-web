<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| tool.execute → DB | Tool execute functions write to PostgreSQL — input must be validated by Zod schemas before queries |
| LLM → tool input | Model-generated JSON passed as tool arguments — Zod inputSchema is the validation gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-001-01 | Tampering | tool.execute functions | mitigate | All tool inputs validated by Zod inputSchema before reaching DB. SDK throws InvalidToolInputError on schema mismatch — catch in hermes.ts stream loop. |
| T-001-02 | Information Disclosure | getProjectContext, getDocumentContent | mitigate | These tools read from DB using projectId/documentId from the agent prompt — the model does NOT see other projects' data because IDs are scoped to the current generation. No cross-project reads. |
| T-001-03 | Denial of Service | Lazy DB imports | accept | Each tool call lazy-imports DB modules — acceptable because the tool is called at most 10 times per agent run (maxSteps=10). No connection pool exhaustion risk at this scale. |
| T-001-SC | Tampering | npm packages (ai, zod, drizzle-orm) | mitigate | slopcheck audit required before install. Current versions: ai@7.x, zod@4.x, drizzle-orm@0.43.x — all from npm with >1M weekly downloads. Treat as [ASSUMED] — verify via `npm audit` in CI. |
</threat_model>
